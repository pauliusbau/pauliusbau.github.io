Dinaminis SQL užklausos "where" sąlygų keitimas

Date an 28 liepa 2015 Tags clause / dynamic / dynamically changing the WHERE clause / oracle / salygu isjungimas / salygu keitimas / sql / where

Situacija dažnai sutinkama kasdieniniame darbe. Turim select'ą su daug where sąlygų. Vieną kartą select'ą tenka įvykdyti su vienomis sąlygomis, antrą kartą su kitomis, trečią - su visomis išskyrus vieną ir pan. Jeigu select'as trumpas, bėdos nėra: užkomentuojam nereikalingas eilutes ir tiek. Jeigu select'as yra labai ilgas, su daug join'ų, union'ų ir daug daug kintančių sąlygų, problema pasidaro gan rimta. Galima lengvai susipainioti ar kažką praleisti. Taip pat toks "užkomentavimo" metodas nelabai tinka automatiniam ataskaitų generavimui. Na bet pasirodo yra gan paprastas ir patogus where sąlygų įjungimo/išjungimo būdas.

Apie viską iš pradžių. Tarkim, turim lentelę t, kurioje yra saugomi įrašai susidedantys iš id, date ir type stulpelių:

 1  ID DT                  TYPE
 2  -- ------------------- -----
 3  1 2014-08-01 00:00:00 A
 4  2 2014-08-01 01:00:00 B
 5  3 2014-08-01 02:00:00 A
 6  4 2014-08-01 03:00:00 B
 7  5 2014-08-01 04:00:00 A
 8  6 2014-08-01 05:00:00 B
 9  7 2014-08-01 06:00:00 A
10  8 2014-08-01 07:00:00 B

Pasibandymui (pav. ant sqlfiddle.com) tokią lentelę galit pasigaminti taip:

 1  create table t
 2  (id number, DT date, type VARCHAR2(5 CHAR));
 3
 4  insert into t (id, DT, type)
 5  SELECT
 6  rownum as id
 7  ,TO_DATE('2014-08-01', 'yyyy-mm-dd') + (rownum-1)*(1/24) AS DT
 8  ,case when mod(rownum,2) = 1 then 'A' else 'B' end as type
 9  FROM dual
10  CONNECT BY LEVEL <= (TO_DATE('2014-08-05 00:00:00', 'yyyy-mm-dd HH24:MI:SS') - TO_DATE('2014-08-01 00:00:00', 'yyyy-mm-dd HH24:MI:SS'))*2
11  order by 1;

Įsivaizduokim situaciją, kad dažnai reikia paselectinti tam tikro laiko visų arba tik 'A' tipo duomenis. Kitaip sakant, dažnai tenka vykdyti arba vieną arba kitą selectą:

1  select \* from t
2  where DT > to_date('2014-08-01 02:00:00', 'yyyy-mm-dd HH24:MI:SS')
3  order by ID

1  select \* from t
2  where DT > to_date('2014-08-01 02:00:00', 'yyyy-mm-dd HH24:MI:SS')
3  and type = 'A'
4  order by ID

Jeigu select'as trumpas, bėdos nėra. Jeigu ilgas ir sudėtingas - atsiranda šiokių tokių problemų. Gerai, kad protingi žmonės sugalvojo genialiai paprastą sprendimą, išnaudojantį loginį ARBA (OR). Naudojantys SQL Developer, galima šį metodą išsibandyti taip:

1  select \* from t
2  where DT > to_date('2014-08-01 02:00:00', 'yyyy-mm-dd HH24:MI:SS')
3  and (1=:ignore_type OR type = 'A')
4  order by ID

Čia, trečiąją kodo eilutę papildžiau išmone 1=:ignore_type. Šią eitulę reiktų suprasti taip: jeigu parametrą ignore_type nustatysime lygų vienam, sąlygą type = 'A' bus ignoruojama. Jeigu nustatysime, kad ignore_type=0 į sąlygą type = 'A' bus atsižvelgta.

Kitaip sakant, parametro ignore_type reikšmę nustatę 1 arba 0 selecto sąlygą vienu ar kitu atveju pakeisime taip:

1  select \* from t
2  where DT > to_date('2014-08-01 02:00:00', 'yyyy-mm-dd HH24:MI:SS')
3  and (1=1 OR type = 'A')
4  order by ID

1  select \* from t
2  where DT > to_date('2014-08-01 02:00:00', 'yyyy-mm-dd HH24:MI:SS')
3  and (1=0 OR type = 'A')
4  order by ID

SQL kalboje išraiška 1=1 yra tolygi TRUE, o 1=0 yra lygi FALSE. O ką daro operatorius OR? Jis pasirenka arba vieną sąlygą arba kitą. Jeigu pirma sąlyga yra teisinga, antroji yra tiesiog ignoruojama. T. y. viršuje aprašyti selectai pavirsta tokiais:

1  select \* from t
2  where DT > to_date('2014-08-01 02:00:00', 'yyyy-mm-dd HH24:MI:SS')
3  and (1=1)
4  order by ID

1  select \* from t
2  where DT > to_date('2014-08-01 02:00:00', 'yyyy-mm-dd HH24:MI:SS')
3  and (type = 'A')
4  order by ID
